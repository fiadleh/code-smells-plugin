Data Clumps are a group of variables that always appears together multiple times in different places. They may have negative impact on code understandability and maintain.
For more info:
DataClump by Martin Fowler
DataClump examples from Wikipedia

Refactoring: Extract the smelly variables to a new class and then refactor all method signatues and calls.

Available Configurations:You can configure it to find dataclumps with a different minimum number of variables or with different hierarchy state.
